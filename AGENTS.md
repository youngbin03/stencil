# AGENTS.md — Stencil

SVG 템플릿 기반 AI 프레젠테이션 생성 시스템. 사양은 `DEVDOC.md`(v4)가 SSOT다. 이 파일은 코드 컨벤션·작업 규약만 다룬다.

## 모델 한 줄
템플릿을 한 번 구워 **디자인 시스템 에셋**으로 만들고, 생성 시 원본을 다시 안 보고 에셋만 조합해 페이지를 만든다(재합성 + 장식 통째 재사용).

## 실행 5단계 (제품 흐름)
① 흡수(SVG→슬롯 분석) → ② 에셋화(토큰·블록·레이아웃·장식 저장) → ③ 구성(Claude, 좌표 없음) → ④ 조립(좌표·피팅·장식 배치) → ⑤ 출력(웹 뷰·다운로드).
①② = 굽기(템플릿당 1회), ③④⑤ = 찍기(생성마다).

## 핵심 설계 불변 (위반 = 설계 위반)
1. 단방향: 흡수→에셋화→구성→조립→출력. 역방향 의존 금지.
2. 생성 시 원본 미조회: 찍기(③④⑤)는 디자인 시스템 에셋만 읽는다.
3. LLM(구성)은 좌표·색·폰트크기를 출력하지 않는다. 레이아웃/블록/콘텐츠만.
4. 결정론: 흡수·조립은 같은 입력 → 같은 출력. 무작위성은 구성(Claude)에만.
5. 장식 보존: 조립은 장식 조각을 변형하지 않는다(재발명 금지).
6. 계약 우선: 단계 간 인터페이스는 `packages/ir` 타입으로만. `ir` 변경은 cross-cutting 결정.

## 패키지 경계 (단계 ↔ 패키지)
- `packages/ir` — DEVDOC 6장 데이터 계약(타입)의 단일 정의처. 모든 패키지가 의존.
- `packages/normalizer` — 흡수 ① : SVG → 슬롯 매니페스트(실측). DONE.
- `packages/extractor` — 에셋화 ② : 매니페스트 → 디자인 시스템 에셋 + 장식 조각. 미구현.
- `packages/composer` — 구성 ③ : Claude tool use → 구성. 미구현.
- `packages/solver` — 조립 ④ : 구성 → 좌표/피팅. DONE(인플레이스 특수케이스).
- `packages/renderer` — 조립 ④ : 렌더 트리 → SVG(장식+텍스트 합성). DONE(인플레이스). 재합성 경로는 Phase 4.
- `apps/web` — 출력 ⑤. 미구현.

## 합성 방식
재합성(에셋 조합)이 v1 기본. 콘텐츠가 원본 슬롯과 1:1로 맞으면 **인플레이스 특수케이스**(default_slots 직접 사용)로 처리. 둘은 같은 조립 단계의 두 경로. 미감 조합 엔진(RCE: 관계 그래프+제약 솔버+비전 비평)은 v2+.

## 입력 자산
- `templates/` = 템플릿 SVG 시드(3테마: black 40 / colorful 32 / green 11, 16:9).
- 텍스트는 실측 `<text>`/`<tspan>`(font-family·size·weight·fill 보존). `data-*` 없음 — 역할은 `id`(Figma 레이어명)로 추론. 일부 id는 콘텐츠 인스턴스명(uncertain 게이트).
- black `Frame-26·30·33`은 `<text>` 없음(장식-only). 31장은 래스터 `<image>` 포함.

## 코드 규약
- 주석·커밋·PR은 영어. 사용자 대면 출력은 한국어.
- 타입은 `ir`에서 import. 단계 내부에서 계약 타입을 재정의하지 않는다.
- 빌드: `npx tsc --build`(루트 프로젝트 참조). PoC 실행: `node scripts/phase1.mjs`.
- 검증 없이 "완료" 선언 금지. Phase 종료 시 `templates/` 샘플로 한 바퀴 회귀.

## 현재 진행
